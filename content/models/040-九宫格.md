# 040. 九宫格

> 编号：040/100 · 分类：问题分析与诊断 · 难度：入门
> 一句话：用九个维度将问题从"一个点"展开为"一个面"——结构化全景扫描，避免盲区。

---

## 一、极简定义

**九宫格（9-Box Grid / Mandala Chart）** 是一种多维度可视化思维工具：将中心问题或主题置于3×3网格中央，周围八个格子分别代表问题的不同维度或关联方面，用于强制性地拓展思维视野，确保不被单一维度束缚。

> 源自今泉浩晃（Hiroaki Imaizumi, 1987）发明的"曼陀罗思考法"（Mandala Chart），是日本企业界广泛使用的创意激发和系统规划工具。

## 二、核心机制

### 2.1 九宫格的四重价值

| 价值维度 | 机制 | 典型产出 |
|---------|------|---------|
| **结构化** | 将模糊问题拆为9个明确子域 | 从"一团乱麻"到"可以一个一个处理" |
| **防盲区** | 强制想到至少8个相关维度 | 防止惯性思维只看到最熟悉的2-3个方面 |
| **可视化** | 一张图上看到全貌和关联 | 便于与团队共享认知、对齐理解 |
| **可操作** | 每个格子都是一个待办项 | 从"想"直接进入"做" |

### 2.2 九宫格的两种布局策略

```
策略A：主题放射型（适合开放式探索）
┌──────────┬──────────┬──────────┐
│ 成本     │ 质量     │ 交期     │
│ 影响     │ 影响     │ 影响     │
├──────────┼──────────┼──────────┤
│ 人员     │ 中心主题 │ 技术     │
│ 影响     │          │ 影响     │
├──────────┼──────────┼──────────┤
│ 客户     │ 供应     │ 法规     │
│ 影响     │ 链影响   │ 影响     │
└──────────┴──────────┴──────────┘

策略B：流程递进型（适合过程诊断）
┌──────────┬──────────┬──────────┐
│ 输入     │ 过程     │ 输出     │
│ （什么进来）│（怎么处理）│（什么出去）│
├──────────┼──────────┼──────────┤
│ 资源     │ 中心主题 │ 反馈     │
│ （需要什么）│          │ （学到什么）│
├──────────┼──────────┼──────────┤
│ 环境     │ 标准     │ 人       │
│ （外部条件）│（判断准则）│ （谁在做）  │
└──────────┴──────────┴──────────┘
```

## 三、理论溯源

- **今泉浩晃（1987）**：《曼陀罗思考法》——将佛教曼陀罗的几何结构转化为商业思维工具，中心是主题，外围八格是相关要素的扩张
- **爱德华·德·波诺**："六顶思考帽"（1985）——多角度平行思考的先驱，九宫格可以视为其空间化扩展
- **战略管理**：麦肯锡的GE/麦肯锡九宫格矩阵——用"行业吸引力×业务竞争力"两个维度构建九个策略区域，是九宫格在战略评估中的经典应用
- **设计思维**：IDEO的"思维导图2.0"——将发散-收敛过程中的发散阶段用九宫格强制量化为至少8个方向

## 四、操作框架

### 4.1 九宫格操作的五个步骤

**步骤1：放置中心问题**
在中央格写下要分析的核心问题/主题。越具体越好——"为什么A产品毛利持续下降"而非"毛利问题"。

**步骤2：选择九宫格类型**
根据问题性质选择放射型还是递进型布局（见上文2.2）。

**步骤3：填充外围八格（第一轮——发散）**
强制填满八个格子，每个格子写一个相关维度或子问题。规则：
- 不允许出现"想不到了"——必须凑够8个
- 不允许两格内容完全重叠——每个格子必须有独立的视角
- 先追求广度，不追求深度

**步骤4：标注优先级（第二轮——收敛）**
对每个格子标注：
- ★★★ = 关键维度（不分析结论出不来）
- ★★ = 重要但非关键
- ★ = 可延后

**步骤5：逐格深度分析**
按优先级从高到低，对每个格子做深度展开（每个格子本身又可以扩展为一个新的九宫格）。

### 4.2 制造业诊断专用九宫格模板

针对制造业/供应链问题的标准化九宫格：

```
┌────────────┬────────────┬────────────┐
│ 原材料      │ 工艺过程    │ 设备状态    │
│ 来源/质量/价格│ 流程/参数/效率│ 健康度/精度/维护│
├────────────┼────────────┼────────────┤
│ 人力        │ 核心问题    │ 质量标准    │
│ 技能/编制/稳定性│            │ 检验/合规/客户要求│
├────────────┼────────────┼────────────┤
│ 成本结构    │ 上下游依赖  │ 外部环境    │
│ 固定/变动/隐性 │ 客户/供应商/物流│ 政策/汇率/竞争│
└────────────┴────────────┴────────────┘
```

## 五、典型应用场景

### 场景1：新产品可行性评估

**中心问题**："是否推出R290工质压缩机？"

放射型九宫格快速展开：
- 技术可行性：实验室数据/量产能力/专利风险
- 市场需求：客户冷链需求/竞品动向/替代威胁
- 成本经济性：BOM成本/规模效应/定价空间
- 法规合规：全球R290法规趋势/认证成本
- 供应链：R290原材料/关键零部件可得性
- 生产准备：产线改造/安全标准/培训
- 销售渠道：现有客户适配/新客户拓展
- 风险评估：安全风险/投资回收期/机会成本

### 场景2：供应商评估与选择

**中心问题**："评估三家候选供应商的综合适配度"

用递进型九宫格：
- 质量：历史合格率/质量体系/客诉记录
- 交期：准时交付率/响应速度/弹性产能
- 成本：报价/隐性成本/付款条件
- 技术：研发能力/工艺水平/协同创新潜力
- 财务：财务健康度/抗风险能力/长期稳定性
- 关系：沟通效率/文化匹配/过往合作经验
- 地理：物流成本/应急响应/地缘风险
- 合规：环境/劳动/行业认证/信息安全

### 场景3：月度经营复盘

**中心问题**："本月经营偏差的根本原因"

- 销售：实际vs预估的订单偏差及原因
- 生产：产能利用率/一次合格率/OEE
- 采购：关键物料价格波动/供应稳定性
- 物流：交付准时率/运输意外/库存周转
- 质量：客诉/退货/过程质量趋势
- 财务：毛利率/现金周转/费用率偏差
- 人力：出勤率/关键岗位稳定性/培训投入
- 客户：满意度/NPS（净推荐值）/客户留存信号

## 六、常见误用与边界

### ❌ 九宫格的陷阱

1. **机械填空**：只满足于"凑够九个格子"而不做深度分析——九宫格是分析起点，不是分析终点
2. **人为割裂**：把九个格子当成彼此独立的孤岛。九宫格的核心价值恰恰在于看到格子之间的联系——成本降低可能影响质量，质量改变反作用于客户满意度
3. **维度僵化**：把某个固定九宫格当成"万能模板"套在所有问题上。好的九宫格维度是因问题而变的，不是因模板而固化的

### ⚠️ 不适用的场景

- **二选一的决策**：二选一用决策矩阵，不需要九宫格
- **深度单维度分析**：如果问题的本质确定在单一维度（如"纯粹是价格问题"），九宫格是在浪费时间
- **极度紧急的危机**：火灾蔓延时不需要九宫格分析火灾原因——先灭火再说
