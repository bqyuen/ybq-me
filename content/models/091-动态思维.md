# 091. 动态思维

> 编号：091/100 · 分类：系统与战略 · 难度：进阶
> 一句话：承认世界是流动的，战略是在行动中持续调整的，而非提前被完整规划的。

---

## 一、极简定义

**动态思维（Dynamic Thinking）** 是一种认知模式：它假设世界本质上处于持续变化中——条件在变、信息在更新、系统在演化——因此"正确"的决策不是一个静态的最优解，而是一个在时间轴上持续调整和适应的过程。动态思维的核心在于"边做边学、边学边调"，而非"分析透彻再行动"。

> 明茨伯格（1994）：战略不是"先规划后执行"的线性过程，而是"涌现"的——在实践中不断成形、不断调整。"深思熟虑的战略"和"涌现的战略"共同构成了"已实现的战略"。

## 二、核心机制

### 2.1 静态思维 vs 动态思维

| | 静态思维 | 动态思维 |
|---|---|---|
| **核心假设** | 世界是可预测的，最优化是在清晰条件下求解 | 世界是持续变化的，"最优"这个概念本身就随时间在移动 |
| **分析方式** | 在某个时间点深度分析所有信息 → 做出一个"正确"决策 | 在时间轴上持续收集信息 → 持续做出"当下最优"的调整 |
| **决策节奏** | "一次性大决定"——计划→执行→检查 | "小步快跑"——行动→观察→调整→再行动 |
| **对错误的处理** | 错误 = 计划失败——需要追责 | 错误 = 信息源——用来更新对世界模型的理解 |
| **适用条件** | 环境稳定、变量有限、因果关系清晰 | 环境动荡、变量众多且相互关联、因果复杂 |
| **典型代表** | 传统年度战略规划（5年计划） | 敏捷开发、精益创业、适应性战略 |

### 2.2 动态思维的三个核心原则

**原则一：承认无知（Epistemic Humility）**
动态思维始于认知谦卑——承认"我目前知道的不够多，不足以做出最优的一步到位的决策"。这不是"不自信"，而是"对复杂性的尊重"。一个优秀的统帅不追求在战前画出完美作战图，而是保有多个选项，在交战后根据对方反应动态选择。

**原则二：决策即实验（Decisions as Experiments）**
每个决策同时也是一个实验——通过执行来获取信息。静态思维问"这是对的吗？"动态思维问"这会教我什么？"如果代价可控，一个"告诉了我重要信息"的失败决策比一个"什么都没教我"的成功决策更有价值。

**原则三：实时反馈闭环（Real-time Feedback Loops）**
动态思维的核心在"反馈"。决策→行动→结果→与预期比较→更新心智模型→下一个决策。这个闭环越快、越短、越真实，动态思维就越有效。反馈延迟是动态思维最大的敌人——延迟让你在已经变化的环境中执行过时的判断。

### 2.3 动态思维的决策框架

```
不是：Plan → Execute（计划 → 执行）
而是：Sense → Interpret → Decide → Act → Sense（感知 → 解释 → 决策 → 行动 → 再感知）
                              ↑_____________________________↓
                                      反馈闭环
```

这个框架的本质是"OODA循环"（观察→定向→决策→行动），由美国空军上校博伊德（John Boyd）在空战战术中提出。博伊德的核心洞察：空战的胜负不取决于谁的飞机更好，而取决于谁的OODA循环更快——你在对手还在"定向"时已经完成了第二轮"行动"。

## 三、理论溯源

- **博伊德（John Boyd, 1970s）**：美国空军战斗机飞行员和军事战略家，提出OODA循环——"观察-定向-决策-行动"的动态决策循环。这一概念从空战渗透到商业战略，成为动态思维的操作框架。
- **明茨伯格（Henry Mintzberg, 1994）**：《战略规划的兴衰》——对传统战略规划的经典批判。"战略不是被设计出来的，是在实践中涌现的。"管理者不是在棋盘前规划全局，而是在湍急的河流中一边划船一边判断方向。
- **精益创业（Eric Ries, 2011）**："构建-测量-学习"的反馈循环——不做完美的产品，而是快速构建最小可行产品（MVP），通过真实市场数据迭代方向。这本质上是动态思维在产品开发领域的应用。
- **敏捷方法论（2001-）**：敏捷宣言——"响应变化胜过遵循计划"。传统瀑布式开发是静态思维（需求分析→设计→编码→测试，一步到位），敏捷是动态思维（迭代开发，每个冲刺后根据反馈调整）。
- **控制论（Wiener, 1948）**：维纳的控制论提供了动态思维的理论基础——系统通过反馈持续修正自身行为，追求的不是"达到终点"而是"保持在目标附近"
- **中国哲学传统**：《孙子兵法》"兵无常势，水无常形"——最高形态的动态思维。不是没有计划，而是计划只是起点，执行中根据敌我变化随时调整。"能因敌变化而取胜者，谓之神。"

## 四、操作框架

### 4.1 建立动态决策习惯

```
步骤1【最小行动】
    → 不是"我要一个完美的方案"，而是"我要一个能让我开始并学到东西的最小行动"
    → 问："我做过的最小版本是什么？它会在哪一点教会我最重要的信息？"

步骤2【设立检测点】
    → 在执行路径上设置明确的、定期的"回头看"点位
    → 问："什么信号出现，我就应该重新考虑方向？"（在行动前定义好"变向标准"）

步骤3【快速提取信息】
    → 每次行动结束后问："我之前不知道但现在知道了什么？"
    → 把"这次学到了什么"本身视为一个关键的产出

步骤4【有勇气变向】
    → 动态思维最难的不是"会调"，而是"敢调"
    → 当证据显示需要变向时，克服沉没成本、"面子成本"、"惯性"
    → 问自己："如果我今天刚接手这个项目，我会走这个方向吗？"
```

### 4.2 动态思维的日常工具

| 工具 | 操作 | 解决什么问题 |
|------|------|------------|
| **前置变向标准** | 在项目启动时写下"如果出现___，我们就应该改变方向" | 防止在应该变向时被"再坚持一下"绑架 |
| **假设日志** | 记录每次决策背后的关键假设，定期检查哪些被证实/证伪 | 让"我们学到了什么"可见、可追踪 |
| **OODA日志** | 重要决策按OODA四步记录：我观察到了什么→我如何解读→我做了什么决定→我得到了什么行动结果 | 看到动态思维的"速度"——从观察到行动用了多久 |
| **小型赌注** | 对不确定的方向，用小规模、低成本的实验取代"大赌注" | 降低"学到的代价"——100万学到的事，用10万也能学到 |
| **事后回顾 vs 事前检验** | 除了事后复盘，也用事前检验（假设项目已经失败，倒推原因） | 在行动前就用动态思维识别可能的变向点 |

### 4.3 经典案例

**案例：Slack的动态转型**
- Slack最初是一家游戏公司（Tiny Speck, 开发游戏 Glitch）
- 游戏没有成功，但团队注意到他们在内部用的通讯工具非常好用
- 动态思维：不是"我们的游戏失败了所以我们失败了"，而是"我们从中学到了一个更有价值的东西"
- 关闭游戏业务，全部资源投向通讯工具 → 成为企业级SaaS巨头
- 没有动态思维的版本：坚持做游戏（"我们已经投了这么多"）→ 烧完钱 → 公司倒闭

**案例：博伊德的OODA在实战**
- 博伊德在空战教学中要求学生"在40秒内将劣势转为优势"
- 关键不在飞得猛，而在于OODA循环比对手快——你的观察-决策-行动速度超过对手的反应速度
- 商业隐喻：不是在同一个战略维度上做得更好，而是决策和执行的速度超越对手

## 五、典型应用场景

### 场景1：月度经营决策

我的月度经营会议：
- ❌ 静态思维：制定年度预算，每月检查"有没有偏离预算"→ 偏离就是"不好"
- ✅ 动态思维：
  - 年度预算是"初始假设"，每月根据新信息更新"滚动预测"
  - 不把"偏离预算"当坏事，而问："外部环境的变化透露了什么新信息？是短期波动还是结构变化？"
  - 每个月都问："如果今天重新制定年度计划，会有不同吗？"

### 场景2：新产品开发

- ❌ 静态：市场调研6个月 → 研发12个月 → 上市 → 发现需求变了 → 三千万打水漂
- ✅ 动态：
  - 第1-2个月：做几个原型给3个客户看（最小行动）
  - 第3-4个月：小批量试产给10个客户试用（建立检测点）
  - 第5个月：根据反馈决定"是加大投入还是转向"（有勇气变向）
  - 如果方向对，6个月开始已经有了客户验证的加持；如果方向错，第3个月就已经止损

### 场景3：供应商谈判

- ❌ 静态：去谈判桌前心里只有一个"理想方案" → 对方不同意 → 僵局或全盘退让
- ✅ 动态：
  - 准备3-5个"可接受的组合"而非1个"最优方案"
  - 谈判中观察对方的反应和限制（实时信息收集）
  - 根据对方暴露的信息动态调整——"原来你最在乎的是付款周期不是价格，那我可以在价格上让一点换付款周期"
  - 谈判本身是信息获取过程——动态思维让你在谈判中变得更"聪明"

## 六、常见误用与边界

### ❌ 对动态思维的误解

1. **动态思维 ≠ 没有计划**：动态思维不是"不用计划，走一步看一步"，而是"有计划，但计划只是第一版假设，不是承诺"。没有计划的动态思维叫"随波逐流"。
2. **动态思维 ≠ 频繁变向**：动态调整不是每周换一个方向——那是混乱。真正动态思维的区别在于：方向不变但路径调整，或证据充分时果断变向。把"犹豫不决"伪装成"动态思维"是常见自欺。
3. **忽视"稳定性"的价值**：不是所有决策都应动态调整——有些基础决策（核心价值观、核心客户承诺、安全底线）需要稳定性。动态的是"如何做"，不是"为什么做"。

### ⚠️ 使用边界

- **反馈周期过长的场景**：有些决策的反馈需要数年才能看到（如教育体系设计、基础科研），OODA循环无法加速——只能通过加强"提前建模"和"多假设同时验证"来模拟
- **不可逆的重大决策**：收购另一家公司、出售核心业务——动态思维的"小步快跑"不适用。这类决策需要最充分的静态分析，因为"做完了就没有调的机会了"
- **过速反应陷阱**：在信息本身就有噪音的环境中（如每日股票涨跌），动态思维如果速度过快就变成了对噪音的过度拟合——每次调整都是在回应随机波动而非真实信号

## 七、与其他模型的关系

| 关系类型 | 模型 | 联动逻辑 |
|---------|------|---------|
| **根模型** | 018.万物系统 | 系统思维告诉我们"世界是关联的"→ 动态思维告诉我们"关联是随时间变化的" |
| **对立模型** | 066.路径依赖 | 路径依赖是"被历史锁死"，动态思维是"每一刻都在重新评估"——打破路径依赖的武器 |
| **对立模型** | 012.笛卡尔思维 | 笛卡尔追求"清晰分明的确定性"，动态思维接受"当下的不完整信息已是最好的起点"——何时该停下来求确定，何时该带着不确定行动 |
| **互补工具** | 054.混沌与秩序 | 混沌理论解释世界不可精确预测，动态思维提供"在不可预测世界中如何决策"的操作方法 |
| **互补工具** | 016.反脆弱 | 动态思维的"小型赌注+快速学习"本身构建了反脆弱——小失败教会你大事情 |
| **互补工具** | 075.飞轮效应 | 一旦飞轮越过临界点，可以适度减少"动态调整"的频率，让飞轮自推——动态思维决定了"调整的频率和幅度" |

## 九、实践检视

**我所在企业 2024-2026 应用案例：**

1. **R290 制冷剂决策（2025）**——动态思维经典应用
   - 静态版本：等 GB 标准完全确定再切换 → 错过 2025 欧盟出口窗口
   - 动态版本：先用 3 台样机跑 6 个月（最小行动）→ 客户反馈"绿色供应链溢价 8%"值得切换 → 2026 全面切换
   - 关键 OODA 加速：把"标准等待"这个反馈周期从"年"压缩到"月"

2. **采购成本谈判（2024）**——动态思维 vs 沉没成本
   - 错误：与某供应商僵持 3 个月，因为"已经投入这么多时间"
   - 修正：每月问"如果今天重新选供应商，我会选它吗？"——答案两次都是"不会" → 第 4 月切换
   - 动态思维教我的：**沉没成本不是方向指标**

**我的 3 条动态思维使用准则**：
- ✅ **每个大决策前写下"变向标准"**——"如果出现 X，我就重新考虑方向"
- ✅ **每月问"如果今天重新规划，方向一样吗"**——给沉没成本一次反弹机会
- ✅ **小型赌注优于大赌注**——100 万学到的，10 万也能学到，且学得更快

**我的 2 个反模式**：
- ❌ **避免在信息噪音大时高频调整**——如股票每日涨跌、供应商每日报价 → 每周看一次
- ❌ **避免在不可逆决策上动态化**——如核心客户、关键团队、长期土地租约 → 静态分析 + 长期承诺

**Q-mapping**：
- 什么时候不该用动态思维？→ 不可逆决策 + 反馈周期数年
- OODA 循环和 PDCA 区别？→ OODA 是"实时对抗"（分钟级），PDCA 是"项目复盘"（周/月级）
- 动态思维和元认知关系？→ 动态思维是"对外变化"，元认知是"对内自我监控"——元认知 监控动态思维的速度是否合适
