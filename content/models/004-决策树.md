# 004. 决策树

> 编号：004/100 · 分类：认知偏差与决策 · 难度：进阶
> 一句话：用树枝状图形把每个选择及其可能结果展开，计算每个路径的期望值，让不确定性变得可见和可算。

重大决策翻车的最常见方式，不是选错了选项，而是**根本没看见某个分支**。会议室里大家争的是"建新厂还是扩老厂"，没人问一句"万一需求崩了呢"。决策树的价值不在于算出那个精确数字，而在于逼你把"万一"画在纸上——看不见的分支没法管理，画出来的分支才可以。

---

## 一、极简定义

**决策树（Decision Tree）** 是一种将决策问题可视化为树形结构的分析工具：每个分支节点代表一个决策点或概率事件，每条路径通向一个最终结果，通过回溯计算期望值来确定最优决策路径。

> 决策树方法起源于20世纪60年代的操作研究（Operations Research）领域，由 Howard Raiffa（1968）在《决策分析：关于选择的导论讲座》中系统化。同一时期，Hunt， Marin & Stone（1966）在机器学习领域独立发展了概念学习系统中的决策树算法。

## 二、核心机制

### 2.1 决策树的三个基本元素

| 元素 | 符号 | 含义 | 示例 |
|------|------|------|------|
| **决策节点** | □（方框） | 你可以控制的选择 | 建新厂 or 扩老厂 |
| **机会节点** | ○（圆圈） | 不可控的概率事件 | 市场需求高/中/低 |
| **结果节点** | △（三角/终点） | 最终收益/成本 | 年利润¥800万 |

分清方框和圆圈，是决策树的第一课，也是最容易被忽视的一课。**你可以控制的是选择，不是结果。**把不可控的东西当成可控（"客户肯定不会跑"），或把可控的东西当成不可控（"市场就这样了，没办法"），是决策质量差的两大根源。

![决策树的三个基本元素](/images/diagrams/004-决策树/1.png)

### 2.2 期望值计算原理

决策树的每一个最终分支都有一个概率和一个结果值：
```
路径期望值 = 概率₁ × 结果₁ + 概率₂ × 结果₂ + ...
```

在机会节点处，取所有分支的加权平均（期望值）；在决策节点处，取期望值最高的分支。

但要记住：**期望值是"重复玩很多次"的平均收益，而你往往只玩一次。**期望值 ¥550 万、但有 10% 概率亏 ¥200 万的方案，对一个现金流紧张的公司可能是致命的——期望值最优 ≠ 你的最优。期望值回答"平均而言哪个好"，不回答"我输得起吗"。

### 2.3 决策树的两种构建方式

| | 前向构建（Forward） | 后向构建（Backward） |
|---|-------------------|---------------------|
| 起点 | 当前状态 | 最终目标 |
| 展开方向 | 决策 → 可能结果 | 目标条件 → 所需决策 |
| 适用 | 已有可选方案，需要评估 | 目标明确，需要规划路径 |
| 分析方向 | 从后往前算期望值 | 从前往后验证可行性 |

无论用哪种方式构建，期望值的计算永远是**从后往前**（逆向递推）——这是决策树分析的铁律。

## 三、理论溯源

- **Raiffa & Schlaifer（1961）**：《应用统计决策理论》将贝叶斯推断和决策分析结合，为决策树提供了数学基础
- **Raiffa（1968）**：《决策分析》一书将决策树推广为商业和管理决策的标准工具，包含大量案例
- **机器学习分支**：Quinlan（1986）发明ID3算法，后发展出C4.5、CART等——决策树不仅是分析工具，也成为了自动从数据中学习决策规则的算法
- **实物期权扩展**：决策树在金融领域的扩展——每个决策节点嵌入了"继续""扩张""放弃"的期权，形成了实物期权分析方法

## 四、操作框架

### 4.1 画一棵决策树的六步法

```
第1步：定义决策问题
    → 用一句话描述："我在____和____之间做选择"

第2步：列出选项和后续事件
    → 每个选项后可能发生什么？（自然状态、市场反应、竞争对手举动）

第3步：估算概率
    → 每个事件发生的概率（用历史数据/专家判断/贝叶斯更新）

第4步：估算每个最终路径的结果值
    → 货币化（利润/成本），或效用值（如果结果不只是钱）

第5步：从后往前计算期望值
    → 机会节点 = Σ(概率×结果)；决策节点 = max(期望值)

第6步：敏感性分析
    → "哪个概率/假设的变化会改变最优决策？" → 确定关键变量
```

六步里最值钱的是第 6 步。概率永远估不准，但你可以算出"概率错到什么程度，结论就会翻转"。如果最优决策在概率 ±20% 的误差范围内都不变，这棵树就够用；如果概率差 5 个百分点结论就翻转，你要做的不是决策，而是先去搞到更好的信息。

### 4.2 简单示例：扩产决策

我的工厂面临决策：建新厂还是扩老厂？

```
决策：建新厂 or 扩老厂？
        ↓
    [建新厂]
    ├── 需求高(60%) → 年利润¥800万
    ├── 需求中(30%) → 年利润¥300万
    └── 需求低(10%) → 年利润-¥200万
    期望值 = 0.6×800 + 0.3×300 + 0.1×(-200) = ¥550万

    [扩老厂]
    ├── 需求高(60%) → 年利润¥450万
    ├── 需求中(30%) → 年利润¥350万
    └── 需求低(10%) → 年利润¥100万
    期望值 = 0.6×450 + 0.3×350 + 0.1×100 = ¥385万

→ 建新厂期望值更高
→ 但注意：低需求时建新厂亏¥200万！风险需单独考虑
```

![扩产决策示例](/images/diagrams/004-决策树/2.png)

再补一刀敏感性分析：需求"高"的概率从 60% 降到多少时，两个方案打平？算出来约 45%。也就是说，只要你对"需求高"的把握低于 45%，扩老厂才是对的。这个数字比期望值本身更能指导行动——它告诉你该去验证什么。

### 4.3 高级技巧：用贝叶斯更新精炼概率

如果我可以花¥10万做市场调研，调研准确率为80%：
- 调研后，用贝叶斯公式更新概率
- 画出"做调研"和"不做调研"的两棵子树
- 计算调研信息的价值 = 有调研的期望值 - 无调研的期望值
- 如果信息的价值 > ¥10万，就值得做调研

这条技巧推广开来就是**信息定价**：任何调研、试点、小批量测试，本质上都是花钱买一棵更准的树。先算信息值多少钱，再决定花多少钱买信息。

### 4.4 分阶段决策：把大决策拆成一串小决策

决策树最高级的用法不是一次算清，而是**把不可逆的大决策拆成一串可逆的小决策**——每一步都保留"扩张/维持/放弃"三个分支。这就是实物期权思维：

- 不一次性投入 ¥2 亿建厂，而是先买地（小钱）、再做设计（小钱）、再动工（大钱）——每个节点都重新评估需求概率
- 不一次性切换全部产品线，而是先做 POC、再小批量、再放量——每一步的失败成本都被限制在可承受范围

## 五、典型应用场景

### 场景1：大客户定价策略
一个大客户要求降价15%，否则"可能"转投竞品。我需要决策。
- 决策节点：降价15% or 不降价
- 机会节点（不降价）：
  - 客户真的转投竞品（70%）→ 损失该客户年利润¥200万
  - 客户是虚张声势（30%）→ 保持原价，年利润不变
- 决策树分析：降价确定损失¥80万，不降价期望损失 = 0.7×(-200) + 0.3×0 = -¥140万
- → 降价损失更小，降价是理性的。但如果"客户转投"真实概率只有40%，则不降价期望损失=¥80万，与降价等价

这个例子还藏着一个行动指向：与其纠结概率是 70% 还是 40%，不如花成本去摸清客户和竞品接触的真实进度——把概率误差从 30 个点压到 10 个点，比任何谈判技巧都值钱。

### 场景2：库存备货决策
旺季前，我要决定某关键物料备多少库存：
- 备货3000件：库存成本¥30万，缺货概率5%
- 备货5000件：库存成本¥50万，缺货概率1%
- 缺货损失：每次缺货导致产线停工损失¥20万
- 决策树+期望值量化"多备货多花的库存费 vs 缺货风险期望值"

### 场景3：自制 vs 外包
某零件：
- 自制：设备投资¥200万 + 单位成本¥8
- 外包：单位成本¥12，无设备投资
- 关键变量：未来3年需求量 → 盈亏平衡需求量 = 200万 ÷ (12-8) = 50万件/3年
- 如果预测需求量>50万件，自制更优

### 场景4：特斯拉超级工厂的分阶段下注（公开案例）

2014 年特斯拉宣布建第一座超级工厂（Gigafactory）时，电动车年销量才几十万辆，没人知道电池需求会不会兑现。一次性赌上几十亿美元建厂，决策树上有一条又粗又黑的"需求不及预期 → 巨额亏损"分支。特斯拉的做法是把这棵树拆开：先与松下合作共建、分阶段投入产能——第一阶段只建部分产线，验证 Model 3 订单量之后再扩产。每一步都保留了"继续/暂停"的决策节点，把一次性的豪赌变成一串有退出选项的小注。这正是决策树的分阶段用法：**不是消除不确定性，而是让自己在不确定性揭晓的过程中始终留在牌桌上。**

### 场景5：西南航空剪掉整片枝叶（公开案例）

决策树每多一个分支，管理成本就翻一倍——不只是算不过来，是运营不过来。西南航空从 1971 年创立起就只飞波音 737 一个机型：一棵树上"机队配置"这个节点只有一条分支。带来的连锁简化是惊人的——飞行员、机务、航材库存、培训体系全部只需一套，飞机日利用率做到行业最高。同期运营多种机型的航司，在每个节点上都有更"灵活"的选择，却为每条分支都付了维护成本。**这个案例的启示是反向用树：有时最优动作不是比较分支，而是剪掉分支——把决策树修成一条直线，用放弃灵活性换取运营效率。**

## 六、常见误用与边界

### ❌ 决策树不适用的情况

1. **概率完全不可知**：黑天鹅事件——你连概率范围都估计不了，决策树只是"精确的错误"
2. **各分支不是互斥的**：如果"需求高"和"需求中"可能同时发生（不同产品线），就不能用简单的概率树
3. **跨期反馈效应**：今天的决策改变明天的概率分布 → 需要更复杂的动态规划或蒙特卡洛模拟
4. **不可量化的价值维度**：品牌声誉、员工士气、战略定位 → 这些难以货币化

### ⚠️ 常见陷阱

- **过度精确**：把概率精确到小数点后两位，但概率的来源只是管理层的"直觉"
- **忽略低概率高影响分支**：10%概率亏损¥1亿 → 期望值只是-¥1000万，但可能需要单独的风险管理
- **未对概率进行贝叶斯更新**：有一份新市场报告出来，但还是用老的概率估计
- **只画不测**：树画完就锁进抽屉。决策树的真正价值在执行中——每个季度对照一次"实际走到了哪个分支"，偏差超过预期就重估概率

## 八、自检清单

```
□ 我的决策节点和机会节点分清楚了吗？（我能控制的 vs 我不能控制的）
□ 概率是如何估计的？来源可靠吗？是否需要贝叶斯更新？
□ 我做了敏感性分析吗？哪个变量的变化会改变最优决策？
□ 低概率高影响的分支，我有没有单独处理？（不只是期望值）
□ 这个大决策能拆成分阶段的小决策吗？每一步有退出选项吗？
□ 我做决策树本身花了多少时间？花的这个时间值不值得？
```

## 九、我实践检视

决策树用多了会发现一个规律：**树的质量不取决于算得多细，而取决于分支列得全不全**。计算人人会，看见没人提的分支才是真功夫。

值得养成的三个习惯：

**会前画树**：凡是投入超过一定金额、或不可逆的决策，开会前先把树画在一张纸上带进会议室。它的作用往往不是得出答案，而是暴露分歧——你会发现争论双方估的根本不是同一个概率，争论立刻从"立场之争"变成"概率之争"，后者才有得谈。

**盯住翻转点**：每棵树算出期望值后，顺手算出"结论翻转的概率阈值"。阈值比结论重要——它告诉你该去收集什么信息、该盯哪个先行指标。

**定期对账**：每半年翻一次过去的决策树，对照实际走到了哪个分支。估偏了不可怕，可怕的是从来不知道自己估偏了——那意味着你的概率校准能力永远不会进步。
